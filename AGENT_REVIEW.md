# Agent Review Protocol

把这个文件交给 Codex、Claude Code、Cursor 或其它 Agent 时，它应该按“审核员”角色工作，而不是直接改模板。目标是判断 `gcc-thesis-template` 是否真的适合提交广州商学院本科毕业论文（设计），并给出有证据的问题清单。

## 审核原则

- 默认只读审核，不修改文件。
- 先列问题，再给总结。
- 所有结论必须有证据：文件路径、行号、命令输出、PDF 页码或截图观察。
- 没有成功编译和视觉检查时，不能说“完全符合学校要求”。
- 如果用户要求“顺手修复”，也必须先输出审核发现，再单独说明修复范围。

## 必读文件

审核前必须读取：

```text
README.md
AGENTS.md

main.tex
styles/gcc-thesis.sty
extraTex/meta.tex
extraTex/front/statement.tex

如果能访问学校原始文件，还要读取或对照：

广州商学院毕业设计（论文）质量标准（试用）.xlsx
```

## 审核命令

先运行：

```bash
python3 scripts/check_structure.py
python3 scripts/doctor.py
```

如果 `doctor.py` 通过，再运行：

```bash
bash scripts/build.sh main
```

如果 `doctor.py` 不通过，不要编造 PDF 结果；只做源码级审核，并明确缺少的工具或 LaTeX 包。

## 审核维度

### 1. 仓库结构

检查：

- `main.tex` 是否生成论文正文。
- `extraTex/` 是否清晰分层。
- `styles/gcc-thesis.sty` 是否只承载格式规则。
- `README.md`、`AGENTS.md`、`docs/compliance-audit.md` 是否没有互相矛盾。

### 2. 学校格式静态对齐

逐项核对：

- A4 纵向打印。
- 页边距：上 2.5 cm、下 2.5 cm、左 2 cm、右 2 cm、左装订线 0.5 cm。
- 中文宋体，英文 Times New Roman。
- 正文小四，1.25 倍行距，首行缩进 2 个汉字。
- 封面无页眉、无页码。
- 封面右上角有 `论文编号：` 和横线。
- 原创性声明和版权授权页无页眉、无页码。
- 封面后、声明后按学校模板保留空白页。
- 摘要、英文摘要、目录用罗马页码，从 `I` 开始。
- 目录本身是否进入目录条目，学校模板应显示 `目录 ... III`。
- 目录只有一页时，目录后是否保留空白页。
- 正文用阿拉伯页码，从 `1` 开始。
- 页眉居中论文题目，右侧广州商学院 Logo，下方横线。
- 章节编号为 `1`、`1.1`、`1.1.1`。
- 正文默认是否保留 5 章结构，且第 5 章为 `总结与展望`。
- `附录`、`参考文献`、`致谢` 是否作为后置一级条目进入目录；默认目录不应额外列出 `附录 A` 小节。
- 图题在图下方，表题在表上方。
- 图、表、公式按章编号。
- 参考文献为 GB/T 7714-2015 顺序编码制。
- 致谢单独成页。
- 附件材料和评分表不是默认审核项；只有用户明确要求时才单独审核。

### 3. PDF 编译与视觉验收

如果编译成功，必须打开 `main.pdf`，至少检查：

```text
main.pdf:
- 第 1 页：封面
- 原创性声明/版权授权页
- 中文摘要页
- 英文摘要页
- 目录页
- 正文第 1 页
- 图表示例页
- 附录页
- 参考文献页
- 致谢页
```

视觉检查重点：

- 字体是否明显异常。
- 页码位置是否居中。
- 页眉 Logo 是否过大、过小或压线。
- 封面字段和横线是否对齐。
- 摘要、目录和正文标题间距是否贴近学校模板。
- 表格是否溢出、压线、跨页异常。
- 参考文献是否能正常打印。

### 4. Agent 工作流可用性

检查：

- README 是否明确说明 Agent-first / Vibe Writing。
- `AGENTS.md` 是否能指导 Agent 修改、编译和验收。
- `skills/gcc-thesis-template/SKILL.md` 是否能作为 Skill 入口。
- `scripts/doctor.py` 是否能给出明确安装建议。
- `scripts/build.sh` 是否支持 `main`、`attachments`、`all`。

## 严重级别

使用下面级别输出问题：

- `P0`：会导致无法编译、PDF 严重缺页、关键学校要求完全缺失。
- `P1`：格式明显不符合学校要求，可能影响提交或评分。
- `P2`：格式或工作流有风险，但可以通过人工复核或小修解决。
- `P3`：文档说明、可维护性、易用性问题。

审核报告必须使用这个结构：

````markdown

- 当前状态：通过 / 部分通过 / 未通过
- 是否完成 PDF 编译：是 / 否
- 是否完成视觉验收：是 / 否
- 不能确认的原因：

## 问题清单

### P0

- [文件:行号 或 PDF 页码] 问题描述。为什么违反学校要求或阻断使用。建议修法。

### P1

- ...

### P2

- ...

### P3

- ...

## 已通过项目

- ...

## 命令结果

```text
粘贴关键命令和结果，不要粘贴无关长日志。
```

## 下一步建议

1. ...
2. ...
````

如果没有发现某一等级的问题，写“未发现”。
